# PVO 最適化フルグリッド計画 (Step2 事前タスク)

目的: Donchian ブレイクアウトの質を向上 (ノイズ削減・初動捕捉最適化) し、ADX導入前に出来高モメンタム(PVO)フィルタの最適パラメータ帯域を確定する。

## 現行設定
- pvo_s_term = 5
- pvo_l_term = 70
- pvo_threshold = 20
課題: 長期EMA=70が平滑過多で反応遅延、閾値20は低ボラ局面でエントリー過度抑制の懸念。

## グリッド定義 (Option 1: フルグリッド)

短期EMA候補 (S): 5, 8, 12  
長期EMA候補 (L): 26, 50, 70  
閾値候補 (T): 0, 10, 20, 30  

総組合せ: 3 * 3 * 4 = 36

### 削減基準 (適用済み)
1. S < L を必須 (短期>=長期は無意味かつ PVO ≈ 0 に近づくため除外)  
2. (L=70 かつ T=30) は極端フィルタでトレード枯渇が想定され除外  
3. 実行途中早期停止条件 (後述) に基づき一部枝刈り可能

除外後組合せ数: 36 - 3(短期>=長期) - 1(L=70,T=30) = 32

## 実行戦略
年別: 2024 → 2025 の順で全32セットを backtest。結果は `report/pvo_runs/<year>/` に以下フォーマットで保存。

ファイル命名: `pvo_S{S}_L{L}_T{T}_{year}_summary.json`  
補助: トレード詳細 `pvo_S{S}_L{L}_T{T}_{year}_trades.json`

## 収集メトリクス
1. total_pnl  
2. profit_factor  
3. max_drawdown (USD) / max_drawdown_pct (計算可能なら)  
4. win_rate  
5. trade_count  
6. avg_hold_bars (バックテスト内部または推定)  
7. pvo_pass_ratio = (PVOで通過したDonchian候補数 / Donchian候補総数)  

Donchian候補総数は: 「Donchianが BUY/SELL どちらかを示したバー」をカウント。該当バーで PVO 閾値超過なら通過。

## スコア計算
正規化 (min-max, 外れ値IQRクリップ) した指標を用いて:  
`score = 0.25*PnL_norm + 0.25*PF_norm + 0.25*(1 - DD_norm) + 0.25*WinRate_norm`

補助分析: 閾値別平均 / 長期EMA別平均 / S/L 比率別効果。

## 早期停止条件 (ブランチ刈り)
同一 (L,T) 固定で S=5,8,12 を連続評価中:  
- 3連続で PnL<0 かつ PF<0.8 なら (L,T) 組削除 → 残りSで実行せず次へ。

## EXIT 相互作用後段タスク
最終上位 3 セットについて `max_hold_bars=0` と `20` を再走し PF, PnL, DD 変化率比較。
