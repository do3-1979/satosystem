## 🔍 ログファイル問題の原因と対策

### 📋 問題の発見

初回実行時、**Q2 2024** と **Q3 2025** で以下のエラーが発生：
```
❌ エラー: 'list' object has no attribute 'get'
```

### 🔎 原因分析

ログファイルが **2つの異なる形式** で混在していました：

#### 1️⃣ **成功したファイル形式**（dict型）
```
backtest_summary_20251209004157.json
├── type: dict
├── keys: ['total_pnl', 'profit_factor', 'max_drawdown', ...]
└── 用途: メトリクスサマリー出力 ✅
```

#### 2️⃣ **失敗したファイル形式**（list型）
```
20251209004149.json
├── type: list (827要素)
├── 要素: [{'close_time', 'position_size', 'profit_and_loss', ...}, ...]
└── 用途: 取引詳細ログ ❌
```

### 🛠️ 対策実施

**修正内容:**

```python
# 修正前：すべてのJSONファイルを対象
log_files = glob.glob('logs/*.json')
latest_log = max(log_files, key=os.path.getctime)

# 修正後：正しいファイル形式を優先的に検索
# 1. backtest_summary_*.json を優先
summary_logs = glob.glob('logs/backtest_summary_*.json')
if summary_logs:
    latest_log = max(summary_logs, key=os.path.getctime)
else:
    # 2. 見つからない場合は詳細ログを使用
    log_files = glob.glob('logs/*.json')
    log_files = [f for f in log_files if 'backtest_summary' not in f]
    latest_log = max(log_files, key=os.path.getctime)

# 3. ファイル形式の検証
if isinstance(data, list):
    print("⚠️  ログファイル形式が古い形式です（list）")
    return None
elif isinstance(data, dict):
    metrics = data  # 正常に処理
```

### ✅ 結果

修正後、**8/8 四半期すべてで成功**：


## 🔧 ログファイル形式の技術仕様

### ファイル名パターン

| パターン | 形式 | 用途 | 優先度 |
|---------|------|------|--------|
| `backtest_summary_*.json` | dict | メトリクス出力 | 🔴 1位 |
| `YYYYMMDDHHMMSS.json` | list | 取引詳細 | 🟡 2位 |

### dict形式（正規形式）

```json
{
  "total_pnl": 971.990217,
  "profit_factor": 2.059569,
  "max_drawdown": 450.422462,
  "max_drawdown_rate": 31.666089,
  "sharpe": 1.108191,
  "win_rate": 100.0,
  "trades": 38,
  "samples": 991
}
```

### list形式（詳細形式）

```json
[
  {
    "close_time": "2024-01-01 00:00:00",
    "position_size": 1000,
    "profit_and_loss": 50.5,
    "decision": "BUY",
    "side": "LONG",
    ...
  },
  ...
]
```
