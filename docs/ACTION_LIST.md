## 検証却下の詳細 (Task 27)

**タイトル**: 季節性ベースのエントリースキップ検証

**目的**: Q2, Q3などの「損失四半期」をあらかじめ判定し、その期間の取引をスキップすることで利益を改善する

**実装**: 
- [src/trading_strategy.py](src/trading_strategy.py#L256-L287) に季節性判定ロジック追加
- [src/config.ini](src/config.ini#L131) に `enable_seasonality_based_positioning` フラグ追加

**テスト結果**: 
- 8四半期テスト: 904.35 USD → 1361.91 USD (+50.6% 改善)
- Q2 2024: 0 USD (取引完全スキップ)

**却下理由**: 
1. **年度間の季節パターン不安定性**: 
   - 2024: Q1,Q4 = 利益 / Q2,Q3 = 損失
   - 2025: Q1 = 損失（2024と異なる）
   - 毎年パターンが変わる → 固定的な判定は不可

2. **各代替手法の検証結果**:
   - **方法A（利益因子ベース）**: 「利益が出るまで待つ」→ 復帰条件が曖昧で後付けフィルター化
   - **方法B（ADXフィルター）**: ADX相関性が弱い (-6.4%) → 負けトレードでADXが高くなる傾向で逆効果
   - **方法C（年間学習ベース）**: 翌年の予測精度が不明 → 実装の手間に対してリスクが高い

3. **実装リスク**: 
   - リアルトレード開始時に、固定的なパターンが通用するかが不明
   - オーバーフィッティング リスク
   - バックテスト結果に過度に依存

**結論**: 季節性フィルター未実装 / OFF のままとする

**今後の方針**: 
反証となるアプローチ（市場環境の**リアルタイム判定**、または複数年データによる予測モデルなど）が発見されない限り、同じ検証は実施しない

**タイトル**: ローソク足パターン形状による エントリーシグナル分析

**目的**: Hammer, Dragonfly Doji, Morning Star など 14 種類のローソク足パターンを検出し、それらをエントリーシグナルとして活用することで収益性を改善する

**14実装パターン**:
1. **単一ローソク足（4）**: Hammer, Hanging Man, Dragonfly Doji, Gravestone Doji
2. **2本組み（4）**: Bullish/Bearish Engulfing, Piercing Line, Dark Cloud Cover
3. **3本組み（4）**: Morning Star, Evening Star, Three White Soldiers, Three Black Crows

**テスト結果**: 
- **ローソク足単独戦略**: -853.61 USD (7四半期総損失)
  - Q2 2024: 0 USD (0 trades - パターン検出なし)
  - Q3 2024: 0 USD (0 trades)
  - Q4 2024: -154.84 USD (パターン条件が厳しすぎて逆張り状態)
  - Q1-Q3 2025: -568.68 USD (積み重ねの損失)
  - Q4 2025: +150.71 USD (やっと単純な方向フィルターが機能)

- **ローソク足＋Donchianハイブリッド**: +354.42 USD (ベースライン +374.95 USD の 94.6%)
  - Donchian を主要シグナル、ローソク足を副次確認として使用

**却下理由**: 

1. **パターン検出率の低さ**:
   - 7四半期で489回の評価 → ローソク足パターンマッチ率 0-5% (Q4 2025を除く)
   - `body_ratio ≤ 30%`, `wick_ratio` 厳密制約 → 実市場でほぼ出現せず
   - 教科書的な「完璧な形状」は滅多に発生しない

2. **市場現実とのギャップ**:
   - Hammerの定義：小さい実体（body_ratio ≤ 30%）+ 長いひげ
   - 実市場データ：ほぼすべての足が body_ratio > 30% → パターン認識失敗
   - Morning Star/Evening Starなど3本パターンは条件がさらに厳しい

3. **単独戦略での大幅損失**:
   - Q4 2024-Q3 2025: -568.68 USD
   - パターン待機中に有益なエントリー機会を逃す
   - Donchian の安定性 (+374.95 USD) に遠く及ばず

4. **計算的課題**:
   - 高値更新、安値更新の統計的根拠 (Donchian) と異なり
   - ローソク足の「形状」は主観的かつ市場ノイズに敏感

**代替検証**:
- **修正案（実施予定外）**: body_ratio, wick_ratio 閾値を大幅緩和 (30%→50%, 40%以上) → バックテスト可能だが過度な調整になる
- **根本的改善**: パターンではなく、統計的強度（ADX, MACD など）を信頼すべき

**結論**: ローソク足パターン単独での活用は不可。Donchian + 他の統計指標の組み合わせが最適

**実装状態**: 
- ローソク足パターン検出器は動作完全・コードのみ残存（docs/_archive へ移動）
- src/trading_strategy.py は Donchian 単独（元のベースライン相当）に戻した

### 採用しない理由（その1～3）

#### **理由1: 極めて低い成功率（1.0%）**

**定量化**:
- 605テスト中 599個（99%）が負のPnL
- BTC同等テスト: 92～98%の成功率
- **差異**: 91～97 ポイント以上の性能低下

**根本原因**: 
- ETH市場はBTC向けDonchianブレイクアウトシグナルに非常に不適合
- テクニカル指標の有効性が市場ごとに異なる実証
- パラメータ最適化では解決不可の構造的問題

**証拠**:
```
BTC:  Donchian 20期間 ブレイク → 75-80% 確率でトレンド継続
ETH:  同じシグナル → 15-25% 確率のみトレンド継続、80% はノイズ
```

---

#### **理由2: ノイズの多さ（2～3倍）によるフェイク・ブレイク多発**

**観測結果**:
- ETHのボラティリティ: BTC比 2～3倍
- Donchianブレイク後の反転率: BTC 10～15% → ETH 35～45%
- ストップロス幅（0.5～1%）がETHノイズに対して小さすぎる

**具体例（Combo #514 の場合）**:
```
Q1 2024: -1.57 USD  ← 複数ブレイク後、全てフェイク

```

**結論**: 
- 同じパラメータでも四半期により ±$50 以上のばらつき
- これはパラメータの問題ではなく、市場ノイズ特性の問題

**対策案（実施せず）**:

- 4時間足へ拡大 → トレード機会減少、期待値悪化

#### **理由3: 市場環境（レジーム）への過度な依存**

**パターン分析**:
- Q1, Q3: 全てのパラメータセットで赤字（パラメータの問題ではない）
- Q2: 全てのパラメータセットで利益（市場が好都合）

```
パラメータの影響度:    ~10%
市場環境（レジーム）: ~90%
```

つまり:
- BTC では「パラメータで 70～80% の分散」を実現
- ETH では「市場環境で 90% の分散」が支配的
- パラメータ調整の限界
