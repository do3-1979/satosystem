# レグレッションテスト方針

本ドキュメントは、satosystemプロジェクトのレグレッションテストの設計・運用方針を定めるものです。

## 1. テストの目的
- 主要機能（バックテスト・ホットテスト・各クラス/メソッド単体・出力整合性）の変更前後での動作・結果の一貫性を保証する。
- 変更による副作用や予期せぬ不具合を早期に検知し、品質を維持する。

## 2. テスト実施ルール
- 合否のみを判定し、失敗時はユーザーへ報告・指示待ちとする（自動修正・対策は行わない）。
- テスト仕様・期待値・判定基準は本ドキュメントに明記し、テストスクリプトはこの方針に従うこと。
- SRC以下の実装とproject_structure.jsonの不整合も検知・報告対象とする。

## 3. 主なテスト項目

### 3.1 バックテスト
- `bot_run.sh` によるバックテストが正常に実行できること
- 変更前後で損益・取引指標・処理時間等の結果が一致すること
- 結果が変化した場合は、想定効果・乖離範囲を明示し、ユーザー指示を待つ

### 3.2 ホットテスト
- `config.ini` の `backtest=0` で180秒間ホットテストを実施
- 60秒ごとの価格取得が2回以上正しく動作すること
- bybit_exchange.pyでダミートレードラッパを用意し、backtest=0時は必ずダミーに切り替える

### 3.3 主要クラス・メソッド一覧テスト
- project_structure.jsonを参照し、各クラス・メソッドの単体テストを作成
- 一括実行し、全てのメソッドが正常動作すること

### 3.4 結果の整合性
- ログ・出力に以下の不整合がないことを確認
    - 指標値の不整合
    - ENTRYしていないのにADD/EXITが記録される
    - ENTRYの多重発生
    - ストップ値の幅が正しい
    - ボラティリティ・PSAR等の計算が正しい
    - 取引不能時は新規取引しない
- 異常時は原因・修正案を提示し、ユーザー指示を待つ

## 4. テスト運用
- テストはdocs/配下にスクリプト・結果を保存し、変更前後の比較が容易な形で管理する。
- テスト仕様の変更・追加は本ドキュメントを必ず更新する。
