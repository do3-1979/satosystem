# ログ出力フローとデータ流

## 1. 全体のデータフロー

```
┌─────────────────────────────────────────────────────────────────────┐
│ Bot.run() - メインループ                                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  for each iteration:                                              │
│    1. OHLCV データ取得 (1分足)                                      │
│    2. 技術指標計算 (Donchian, PVO, ATR, Volatility, etc)          │
│    3. トレード判定                                                  │
│    4. リスク管理                                                    │
│    5. ポートフォリオ更新                                             │
│    6. [★ ここでログ出力 ★]                                          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. ログ出力の制御フロー

```
┌────────────────────────────────────────────────────────────────┐
│ Bot.__init__() で初期化                                         │
├────────────────────────────────────────────────────────────────┤
│  self._log_counter = 0                                        │
│  self._logging_interval = Config.get_logging_interval()      │
│                                                              │
│  # デフォルト: logging_interval = 10000                      │
└────────────────────────────────────────────────────────────────┘
                          ↓
┌────────────────────────────────────────────────────────────────┐
│ メインループの毎イテレーション                                   │
├────────────────────────────────────────────────────────────────┤
│  self._log_counter += 1                                       │
│                                                              │
│  # ログ出力判定                                               │
│  force_log = (trade_executed) OR                             │
│              (self._log_counter % self._logging_interval == 0)│
│                                                              │
│  if force_log:                                               │
│      logger.log_trade_data(...)  # ログに記録                 │
│      pnl_history.append(...)     # PnL履歴に記録              │
└────────────────────────────────────────────────────────────────┘
```

---

## 3. ログ出力の判定ロジック

```python
# src/bot.py L725-730

self._log_counter += 1

# 以下のいずれかで常にログ出力:
force_log = (trade_executed)           # ★ トレード実行時 (重要イベント)
            or                         #
            (self._log_counter %       # または
             self._logging_interval    # logging_interval の倍数
             == 0)

if force_log:
    # ログ出力処理
```

**重要**: トレード実行（ENTRY/ADD/EXIT）時は **logging_interval に関わらず常にログ出力**

## 4. データ出力の流れ

```
┌─────────────────────┐
│ Bot.run()           │ 各イテレーション
└──────────┬──────────┘
           │
           ├─→ [ログ出力判定] force_log?
           │        │
           │   ┌────┴────┐
           │   │ YES     │ NO
           │   ↓         ↓
           │  ┌────────┐ [スキップ]
           │  │Logger.│
           │  │ log() │
           │  └────┬───┘
           │       │
           ├─→ [メモリバッファ] in-memory list
           │       │
           └─→ [ファイル出力] JSON/ZIP (ログファイルクローズ時)
                    │
                    └─→ logs/YYYYMMDDHHMMSS.json
                    └─→ logs/YYYYMMDDHHMMSS.zip (圧縮)
```

## 6. ログ間引きの影響

```
logging_interval = 10000 の場合（デフォルト）:

    イテレーション数: 978本

    ┌─ 1    ┐      ┌─ 2000  ┐      ┌─ 10000 ┐
    │ スキップ│ ...  │ スキップ│ ...  │ [ログ] │
    └────────┘      └────────┘      └────────┘

    結果: 978本のうち 0本だけログ出力
    → グラフに全く指標が表示されない ✗


logging_interval = 1 の場合（全ログ）:

    ┌─ 1    ┐ ┌─ 2    ┐ ┌─ 3    ┐ ... ┌─ 978  ┐
    │ [ログ] │ │ [ログ] │ │ [ログ] │     │ [ログ] │
    └────────┘ └────────┘ └────────┘     └────────┘

    結果: 978本全てログ出力
    → グラフに全指標が正確に表示 ✓
```

## 10. よくある質問

**Q: なぜログ間引きが必要？**
A: 毎イテレーション ログ出力 (logging_interval=1) だと処理が30%遅くなり、ファイルサイズも1000倍になります。本番テストでは logging_interval=10000 で高速化します。

**Q: グラフで指標が正確に見えない？**
A: logging_interval が高いと、データ点が飛び飛びになり、線グラフが正確でなくなります。グラフ分析には必ず logging_interval=1 を使用してください。

**Q: どの設定を使えばいい？**
A: グラフ分析 → logging_interval=1、本番テスト → logging_interval=10000、バランス → logging_interval=100
