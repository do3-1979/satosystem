## 1. システム概要

### 1.1 戦略名・アルゴリズム

**Donchian Breakout + PVO + ADX フィルタ + Two-Tier Entry**

BTC/USDT 無期限先物（4H足）のトレンドフォロー型自動売買システム。  
ドンチャンチャネルのブレイクアウトで方向を検出し、PVO（出来高トレンド）・ADX（トレンド強度）・TSMOM（モメンタム）の3フィルタで偽ブレイクを除外する。

### 1.2 取引所分離アーキテクチャ

| 役割 | 取引所 | 用途 |
|------|--------|------|
| データ取得 | Bybit（公開API） | OHLCV・Ticker・シグナル生成 |
| 注文実行 | Bitget | エントリー・エグジット（本番/ダミー） |

### 1.3 コアモジュール構成

| モジュール | 責務 |
|-----------|------|
| `bot.py` | メインループ・注文執行・PnL管理 |
| `trading_strategy.py` | ENTRY/ADD/EXIT 判断ロジック |
| `risk_management.py` | ポジションサイジング・PSAR/ADX/ドンチャン計算 |
| `exit_strategy_v2.py` | 多段階エグジット（PSAR/Chandelier/PSL/VolClimax/Composite） |
| `price_data_management.py` | OHLCVバッファ・シグナル生成・時間進行 |
| `portfolio.py` | ポジション追跡・損益履歴・ドローダウン |
| `config.py` | 設定一元管理（config.ini + .api_key） |
| `ohlcv_cache.py` | SQLiteベースOHLCVキャッシュ（高速バックテスト） |
| `risk_overlay.py` | DDキルスイッチ（DD/日次/連敗停止、デフォルト無効） |
| `cost_model.py` | 手数料・スリッページモデル（バックテスト用、デフォルト無効） |

## 3. エントリー/エグジット条件

### 3.1 エントリー条件（全て AND で成立時のみ）

```
1. ポジション未保有（position_qty ≤ 0）
2. ドンチャンブレイクアウト
   - BUY:  close > Donchian High（直近30本の最高値）
   - SELL: close < Donchian Low（直近30本の最安値）
3. PVO フィルタ: PVO > 10
   - PVO = ((EMA5_volume - EMA70_volume) / EMA70_volume) × 100
4. ADX フィルタ: ADX > adx_filter_threshold（31）
5. TSMOM フィルタ: モメンタム方向と一致（150本ルックバック）
6. 週末フィルタ: 土日はエントリーしない
```

### 3.2 Two-Tier Entry System（ピラミッディング）

| Tier | 条件 | サイズ |
|------|------|--------|
| Tier 1 | 初回ブレイクアウト成立時 | 100% |
| Tier 2 | 追加エントリー条件成立時（entry_range ATR以内） | entry_times=2 |

### 3.3 エグジット条件（優先順）

| 優先度 | トリガー | 条件 | 状態 |
|--------|---------|------|------|
| 1 | **PSAR トレーリングストップ** | BUY: close ≤ PSAR / SELL: close ≥ PSAR | **有効（メイン）** |
| 2 | **シャンデリアエグジット** | BUY: close < (22期間高値 - 3.0×ATR) | 無効（BTC） |
| 3 | **時間ベースエグジット** | 保有時間 > max_holding_hours | 無効（BTC） |
| 4 | **Profit Step Lock** | MFE ≥ 2% → 1%ロック / ≥ 4% → 2.5% / ≥ 8% → 6% | オプション |
| 5 | **Volume Climax** | 出来高 > 20本平均 × 3.0 かつ利益 ≥ 0.5% | オプション |
| 6 | **Composite Score** | ADX下落5pt + PVO反転 + 出来高枯渇 → スコア≥2 | オプション |
