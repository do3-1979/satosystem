# Task 40j - ホット（ダミー） vs バックテスト 差分検証（仕様差として許容）

- 日付: 2026-02-03
- タスク: 40j
- 結論: **現時点は仕様差として許容し、ロジック修正は行わない**（優先タスク39d等を先行）

## 背景
ホットテスト（ダミー）は周期更新（例: 60秒）で最新足を更新しながらシグナル計算する一方、バックテストは1本足単位で進むため、
足の確定タイミング/参照インデックス/状態更新の差でシグナル一致しない可能性がある。

## 実施内容（40j）
- 差分再現（ホット相当/バックテスト相当でDonchian/PVOの一致を確認し、不一致なら差分を出力）
  - スクリプト: `test/test_hot_backtest_parity_regression.py`
- 「1本（以上）遅延」が損益に与える影響を実データで測定
  - スクリプト: `test/test_signal_delay_pnl_impact.py`
  - 入力データ: `ohlcv_data/ohlcv_cache.db` の BTC/USDT:USDT 4時間足（2025Q4: 551本）
  - モデル: シグナルは確定足系列で計算し、約定価格だけを lag 本後ろにずらして比較

## 観測結果（損益感度の一例）
`test/test_signal_delay_pnl_impact.py` 実行結果（2025Q4, 4h）:

- Donchianのみ（フィルタなし）
  - lag0: pnl=-26326.60, trades=9
  - lag1: pnl=-21955.50, trades=9
  - diff(lag1-lag0): **+4371.10**

- Donchian + PVO（閾値適用）
  - lag0: pnl=+14718.50, trades=2
  - lag1: pnl=+13689.20, trades=2
  - diff(lag1-lag0): **-1029.30**

※ 結果は期間/条件で符号が変わり得る（遅延が常に良い/悪いとは限らない）。

## 判断（仕様差として許容）
- 差分は再現できたが、ここでは「ロジック不具合の修正」ではなく「差分の存在と損益感度の把握」が目的。
- 40jは**検証タスクとして完了**し、優先度の高い利益拡大タスク（39d等）を先行する。
