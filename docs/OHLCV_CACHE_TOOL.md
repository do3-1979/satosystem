# OHLCV キャッシュ検査ツール

OHLCVキャッシュ（`ohlcv_cache.db`）の内容を確認・管理するツールです。

## 機能

- 🗄️ **キャッシュサマリー表示**: 総レコード数、ファイルサイズ、パラメータ一覧
- 📈 **データ範囲分析**: 取得データの期間、セグメント（連続したデータ）を表示
- ⚠️ **断絶検出**: データに途中ギャップがある場合、そのギャップを表示
- 🔬 **詳細分析**: すべてのパラメータについて詳細情報を表示

## 使用方法

### 1. キャッシュサマリーを表示（デフォルト）

```bash

```


### 3. 詳細分析をすべて表示

```bash
./ohlcv_cache_info.sh all
```

すべてのパラメータについて詳細分析を表示します。複数のタイムフレームでキャッシュがある場合、各タイムフレームごとにセグメント情報が表示されます。

### 4. ヘルプを表示

```bash
./ohlcv_cache_info.sh help
```

## 直接Python実行

より詳細な制御が必要な場合は、Pythonスクリプトを直接実行できます：

```bash
cd src
python ohlcv_cache_inspector.py --help
```

### 利用可能なオプション

- `--summary`: キャッシュサマリーを表示
- `--coverage`: データ範囲と断絶を表示
- `--all`: 詳細分析をすべて表示
- `--start EPOCH`: 開始時刻（エポック秒）を指定
- `--end EPOCH`: 終了時刻（エポック秒）を指定
- `--timeframe MINUTES`: タイムフレーム（分）を指定
- `--cache PATH`: キャッシュファイルのパスを指定（デフォルト: `ohlcv_data/ohlcv_cache.db`）

## キャッシュの仕組み

1. **初回実行**: BybitサーバからデータをダウンロードしてSQLiteキャッシュに保存
2. **次回以降**: キャッシュから取得して使用（高速）
3. **蓄積**: バックテストを実行するたびにキャッシュが蓄積される

### キャッシュの構造

- **テーブル名**: `candles`
- **主なカラム**: 
  - `start_epoch`, `end_epoch`, `time_frame`: パラメータ
  - `close_time`, `close_time_dt`: 時刻情報
  - `open_price`, `high_price`, `low_price`, `close_price`: OHLC値
  - `volume`: 取引量

### キャッシュをクリアしたい場合

```bash
# Pythonで直接実行
cd src
python -c "from ohlcv_cache import OHLCVCache; cache = OHLCVCache(); cache.clear_cache()"
```

または、ファイルを削除：

```bash
rm src/ohlcv_data/ohlcv_cache.db
```

## トラブルシューティング

### キャッシュファイルが見つからない

```
❌ キャッシュファイルが見つかりません
```

解決方法:
- バックテストを一度実行してキャッシュを生成してください
- または、`src/ohlcv_data/` ディレクトリが存在することを確認してください

### tabulateライブラリがない

```
ModuleNotFoundError: No module named 'tabulate'
```

解決方法:
```bash
pip install tabulate
```
