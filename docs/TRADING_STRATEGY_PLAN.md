## 1. Phase 1マーケットレジーム検出の現状評価

### 🎯 概要

Phase 1マーケットレジーム検出は、市場のボラティリティとトレンド強度から現在のレジームを分類し、エントリー判定を動的に調整する機能です。

### 📊 2年間バックテスト結果

| 期間 | Baseline | Phase 1 | 改善率 | 評価 |
|------|----------|---------|--------|------|
| 2024 Q1 | -$3,541 | -$3,130 | **+11.6%** ✅ | 強トレンド初期 |
| 2024 Q2 | -$295 | -$294 | +0.3% | 効果小 |
| 2024 Q3 | -$356 | -$360 | -1.1% | 効果なし |
| 2024 Q4 | -$316 | -$317 | -0.5% | 効果小 |
| 2025 Q1 | -$418 | -$419 | -0.1% | ほぼ変化なし |
| **2025 Q2** | -$291 | -$127 | **+56.4%** ✅✅ | **保合い→転換** |
| **2025 Q4E** | -$192 | -$260 | **-34.9%** ❌ | **継続トレンド** |

### 🔑 主要発見

#### ✅ 有効な環境

**1. 保合い→トレンド転換時 (2025 Q2: +56.4%)**
- 環境: 4月低ボラ保合い → 5月-6月トレンド転換
- 効果: Win Rate 2.8% → 7.1%, Profit Factor 6倍向上
- メカニズム: 保合い時のSIDEWAYS判定でエントリー制限、転換点で選別

**2. トレンド初期段階 (2024 Q1: +11.6%)**
- 環境: 1月急速トレンド開始
- 効果: 確度の高いエントリー選別による損失削減
- メカニズム: ボラティリティ段階的上昇時の判定精度向上

#### ❌ 非効的な環境

**継続的なトレンド相場 (2025 Q4初期: -34.9%)**
- 問題: ボラティリティが1.2倍に達しない状況で過度にエントリーを制限
- 現象: Win Rate 46.4% → 29.3% (機会損失)
- 根本原因: STRONG_TREND判定基準（volatility_ratio >= 1.2）が実現困難

## 3. Phase 2: 段階的フィルタリング実装（2025-11-24完了）

### ✅ 実装内容

**Binary ブロック → Graduated Sizing 移行**

```python
# Phase 1: Binary (ブロック)
if regime == SIDEWAYS:
    decision = NONE  # 取引完全ブロック

# Phase 2: Graduated (段階的)
if regime == SIDEWAYS:
    position_size = base * 0.75   # リスク削減
elif regime == WEAK_TREND:
    position_size = base * 1.0    # 基準
elif regime == STRONG_TREND:
    position_size = base * 1.25   # 積極的
```

### 🚨 重大課題：全期間で赤字

#### 現象
```
2024年:
  Q1: -118,027 USD (96.83% 勝率)      ← 勝率高いのに大損失？
  Q2: -9,817 USD   (13.64% 勝率)
  Q3: -11,859 USD  (0.00% 勝率)
  Q4: -10,532 USD  (6.25% 勝率)
  平均: -37,559 USD

2025年:
  Q1: -13,940 USD  (92.31% 勝率)      ← 同じく矛盾
  Q2: -9,699 USD   (2.78% 勝率)
  Q3: -9,981 USD   (0.00% 勝率)
  Q4初期: -6,416 USD (46.38% 勝率)
  平均: -10,756 USD
```

#### 推定原因

1. **ポジションサイズロジック異常**
   - 損失が初期資本 10,000 USD の **3～12倍** に達している
   - レバレッジなし設定（leverage=1）なのにこの損失は物理的におかしい
   - 推定: ロング/ショートの切り替え時に累積損失がカウントされている？

2. **ストップロス機能不動作**
   - Max DD が常に 0 になっている
   - PSAR（放物線SAR）や trailing stop が有効に動作していない
   - トレーリング最大マージンが実装されていない可能性

3. **シグナル品質と収益性の乖離**
   - 高勝率（96.83%, 92.31%）なのに大損失
   - → 利益トレード < 損失トレード という構図
   - → トレード毎のサイズが動的に変動している可能性

4. **スリッページ・手数料未考慮**
   - バックテストが理想的な約定を仮定
   - 実際のマーケット環境では滑るコストを無視
