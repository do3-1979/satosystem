# トレンドフォロー戦略 評価指標設計

本戦略は「トレンド発生時の利益伸長」と「騙し(フェイク/ノイズ)時の損失極小化」を重視する。従来の勝率・PFのみでは“伸ばす力”と“避ける力”が十分に可視化できないため、以下の補助指標を導入する。

## 1. トレード粒度指標
| 指標 | 目的 | 定義 / 計算 |
|------|------|-------------|
| MFE (Max Favorable Excursion) | 利益伸長余地 | エントリー後の最大含み益値 (価格差 or PnL) |
| MAE (Max Adverse Excursion) | 損失拡大リスク | エントリー後の最大含み損値 |
| 利用率 (Profit Capture Ratio) | 伸ばし取りこぼし度 | 実現利益 / MFE (0〜1) |
| 損失抑制率 (Loss Containment Ratio) | 騙し回避度 | 実現損失 / MAE (損失トレードのみ、0〜1。低いほど良い) |
| トレンド持続バー数 | 滞留効果 | トレンド判定期間中保持したバー数 |
| Exit Give-back (利益の戻し幅) | 早期利確/遅延評価 | MFE - 実現利益 (大きいほど機会損失) |

## 2. 集計レベル指標
| 指標 | 目的 | 定義 / コメント |
|------|------|----------------|
| MFE中央値 / 上位四分位 | 平均的な“伸び”と大トレンドの分布把握 | 全トレード MFE 分布統計 |
| MAE中央値 | 初期逆行の許容度 | 初期リスク特性確認 |
| Capture Ratio 平均 | 利益伸長効率 | 高値付近での最終利確成功度 |
| False Break Loss Median | 騙し発生時損失の標準的規模 | “トレンド失敗”分類トレードのみ損失中央値 |
| Tail Ratio (95%利益 / 95%損失の絶対値) | 分布の歪み (トレンド集中度) | 利益尾部が損失尾部をどの程度上回るか |
| Run-up Preservation (%) | 利益保持力 | (実現利益合計) / (MFE合計) * 100 |
| MAE/MFE 比率 | リスク効率 | (平均MAE) / (平均MFE) (低いほど初期逆行が小さい) |
| Trend Win Quality | 質的勝率 | トレンド分類勝ちトレード中 上位 capture ratio >0.7 の割合 |

## 3. トレンド / 騙し分類ロジック（暫定案）
1. エントリー後 `L` バー以内に価格が `ATR * k1` 以上進行し高値(または安値)を更新 → トレンド候補。
2. その後最大振れ幅 (MFE) が `ATR * k2` 以上継続 → トレンド確定。
3. 確定条件未達で `MAE` が `ATR * k3` まで到達し損失確定 → 騙し分類。
- 推奨初期パラメータ例: `k1=1.0, k2=2.0, k3=1.2, L=20` (調整可能)

## 4. 実装方針
- `order.py` または 約定生成時にトレードID単位データ構造拡張: `{entry_ts, exit_ts, entry_price, exit_price, mfe, mae, classification, capture_ratio, loss_containment_ratio}`
- 更新タイミング: 各バーごとに保有トレードへ高値/安値チェックで MFE / MAE 更新。
- 終了時: 実現PnL計算後 Capture/Loss containment 算出。
- 集計: 新しいレポート生成モジュール (例: `trend_metrics_reporter.py`) で分布統計出力。

## 5. レポート出力追加案
Markdown表 + 主要分布(ヒストグラム) CSV/JSON:
- `trend_trade_metrics.json` : 全トレード行指標
- `trend_summary.json` : 上記集計指標
- 可視化: MFE vs Capture Ratio 散布図, MAE ヒスト, Classification別 PnL 箱ひげ。

## 6. 導入ステップ
1. データ構造拡張 (トレードオブジェクトに MFE/MAE/capture等フィールド追加)
2. ランタイム更新ロジック実装 (保有中トレード毎バー更新)
3. 騙し/トレンド分類条件実装 & フラグ確定時刻記録
4. レポート生成 & 既存 backtest_summary へ一部指標統合
5. パラメータ(k1,k2,k3,L) の Grid / 手動調整評価

## 7. 解釈指針
- Capture Ratio が高く Tail Ratio > 1.5 維持: トレンド伸長捕捉が有効。
- False Break Loss Median が減少しつつ PF 改善: 騙しフィルタ向上。
- MAE/MFE 比率低下: 初期逆行耐性を保ちながらトレンド伸長が効率化。
