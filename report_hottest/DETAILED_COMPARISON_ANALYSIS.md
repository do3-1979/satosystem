### ログ出力行数の大きな乖離

| 項目 | ローカル | ラズパイ | 差異 |

### 原因分析

**ローカル（バックテスト）**:
- 2時間足（time_frame=120）でのみ評価
- 1キャンドル = 1行のログ
- 53キャンドル × 1 = 53行

**ラズパイ（ホットテスト）**:
- リアルタイム1分ごとにログ出力
- 1分 = 1行のログ
- 108時間 × 60分/時間 ≈ 6,480分 ≈ 6,413行

## 4. **重大な問題: シグナルロジックの違い**

### ローカル（バックテスト）の場合

```
期間: 2025/12/08 23:00 ～ 2025/12/13 09:00（5日間）
キャンドル数: 53個（2時間足）
ENTRY検出: 2回
EXIT検出: 2回

→ 市場環境が異なる期間のみ評価
```

**ローカルで取引が発生した理由**:
1. 過去データ（2025/12/08）から開始
2. ドンチャン + PVO シグナルが発生
3. 2 エントリー → 0勝2敗

### ラズパイ（ホットテスト）の場合

```
期間: 2025/12/09 02:13 ～ 2025/12/13 14:14（4.5日間）
1分ごと評価: 6,413回
ENTRY検出: 0回
EXIT検出: 0回

→ シグナル一度も発生なし
```

## 5. **根本原因の仮説**

### 仮説1: データソース の違い

**ローカルバックテスト**:
- Bybitサーバから新たにデータ取得
- 過去データ（2025/12/08以降）を評価
- 確率的にシグナル発生の可能性高い

**ラズパイホットテスト**:
- キャッシュされたOHLCVデータを使用？
- 1分ごとの更新ロジック
- リアルタイムデータは利用不可？

### 仮説2: タイムフレームの違い

**ローカル**:
- 2時間足（time_frame=120）での評価
- 53キャンドル（=106時間分）のデータ

**ラズパイ**:
- ホットテストは1分ごとに評価？
- しかし1分足では取引ロジックが機能しない
- ドンチャン、PVO計算が異なる

### 仮説3: Donchian / PVO パラメータの計算タイミング

```python
# バックテスト（2時間足）
donchian_lookback = 20  # 20 × 2時間 = 40時間
pvo = (12-26-9)        # 12/26/9 × 2時間

# ホットテスト（1分足？）
donchian_lookback = 20  # 20 × 1分 = 20分（短すぎる）
pvo = (12-26-9)        # 12/26/9 × 1分（計算不足）
```
