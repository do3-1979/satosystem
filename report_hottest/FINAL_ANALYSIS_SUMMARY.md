## 分析結果

### 原因仮説 1: **タイムフレームの計算矛盾**

**ローカルバックテスト**（正常）:
```python
# 2時間足で評価
time_frame = 120  # 分単位
donchian_lookback = 20  # 20キャンドル × 120分 = 40時間
pvo_periods = (12, 26, 9)  # 12キャンドル × 120分 = 24時間
```

**ラズパイホットテスト**（疑い）:
```python
# 1分足で受け取る？
time_frame = 1分  # リアルタイム
donchian_lookback = 20  # 20キャンドル × 1分 = 20分（←短すぎる！）
pvo_periods = (12, 26, 9)  # 12キャンドル × 1分 = 12分（←短すぎる！）
```

**結論**: ホットテストが1分足でシグナル評価している可能性があり、ドンチャン計算が実質無効化されている可能性。

### 原因仮説 2: **データ更新タイミング の違い**

**ローカルバックテスト**:
- 過去データを一括取得
- 確定した2時間足でのシグナル判定
- ブレイク判定が「確実」

**ラズパイホットテスト**:
- 1分ごとのリアルタイムデータ受け取り？
- 2時間足が「確定していない」段階でのシグナル判定？
- ブレイク判定が「曖昧」になる可能性
