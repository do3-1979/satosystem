## 現在の基本構成
- 市場: `BTC/USD` 2時間足バックテスト高速化版
- コア: Donchian ブレイクアウト + PVO フィルタ (標準 12/26/0)
- リスク: レバレッジ 1倍 / `risk_percentage=0.03` / ATRベース初期ストップ
- ピラミッディング: `entry_times=4` (最適化結果) / 価格変動幅条件のみ簡易運用
- 出口: PSARトレイリング + ハードストップ / 利食いはストップベース
- 部分利確: 有効化 (閾値 `partial_exit_profit_rate=0.10`, 比率 `0.50`)
- 分類: MFE/MAE vs ATR (採用閾値 `k2=1.5`, `k3=1.2`) → TREND / FALSE_BREAK / NEUTRAL

## 有効化中コンポーネント詳細
### エントリー
1. Donchian 高値/安値ブレイク検出
2. PVO > 0 を最終フィルタ (低ボラ横ばい回避)
3. Keltner は現状無効 (`keltner_enabled=False`) → 検証待ち

### ピラミッディング
- 初回 + 追加 (ADD) を 4回上限 (内部カウンタ) に制限
- ADD条件: 有利方向への価格伸長 + 内部距離しきい値
- 未実装(今後): 陽線/陰線連続カウント・ATR利益進捗条件

### 出口
- EXIT: PSAR反転 or ストップヒットで全決済
- PARTIAL_EXIT: 一定利益率到達後に比率決済 (現在は利益率 & 保持バー条件)
- 未実装(今後): ADX低下 / ボラ急縮小による早期縮小 / 残ポジ再拡張条件

### 分類指標記録
- 各トレードで MFE/MAE/ATR と損益を記録し分類 → 再最適化用データは `trend_trades_*.json`
- 最新グリッド探索結果 (2024年通年): k2=1.5, k3=1.0〜1.5 がほぼ同等トレンド捕捉率 (TrendPnL比 ~54.7%)

### ログ/集計パイプライン
- 実行: `./bot_run.sh run` (APIキー注入/掃除/サマリ一括管理)
- 直接 `python bot.py` 実行時は警告を出し再発防止 (環境変数 `BOT_WRAPPER_INVOKED=1` で抑制可能)
- 出力: `report/backtest_summary_*.json`, `trend_trades_*.json`, `pnl_timeseries_*.json`, 可視化HTML, Excel集計

## 直近改善ロードマップ (優先度順)
1. 部分利確ロジック拡張
   - ATR倍率 & 最低バー & ADXトレンド強度条件導入
   - 段階: 10%→15%→トレール連動比較
2. トレンド分類動的更新
   - 月次で MFE/MAE 分布再計算 → k2/k3 自動再推奨
   - 阈値ドリフト監視 (±10%) 超過時のみ再最適化実行
3. EXIT 改善 (損失抑制)
   - PSAR + ATRトレール複合 / 急速ボラ縮小時早期縮小
   - コア: 「含み益最大時からの回帰率」指標化して閾値適用
4. ADD 精緻化
   - 有利方向伸長 = ATR倍率 + 連続足形判定 + ADX水準
   - 追加ごとにサイズ逓減 (例: 初回100% → 60% → 40% → 30%)
5. Keltner再評価
   - A/B (enabled vs disabled) を 2024Q2/Q3 月次で比較
   - 指標: PF / トレード数 / 平均MFE / 初期MAE
6. 部分利確データ活用
   - 部分利確後残ポジ平均保持バー / 追加利益率 / 再反転損失率収集
   - 収集後「二段階利確」(50%→25%) 試験
7. リスク再構成
   - 最大同時ポジション総ATRエクスポージャー限界 (例: Σ(ATR*数量) <= 資本×X%)

## 2025年パフォーマンス劣化分析結果 (2025-11-21)

### 月次バックテスト結果サマリ
- **2024年全体**: PnL +$125.43 (+41.8%), 勝率 97.80%, 勝ち月 7/12
- **2025年 1-10月**: PnL -$299.02 (-70.3%), 勝率 1.92%, 勝ち月 2/10
- **累積結果**: 初期 $300 → 最終 $126.41 (-57.86%)

### 根本原因
固定閾値 (k2=2.2, k3=1.6) が2024年トレンド市場で最適化されたが、2025年レンジ相場には不適合。市場レジーム変化への適応機能なし。

### 教訓
静的パラメータは市場環境変化に脆弱。適応型システム導入が必須。
