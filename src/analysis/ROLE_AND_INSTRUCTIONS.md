# 役割と運用指示 (GitHub Copilot エージェント)

最終更新: 2025-11-16

## 役割
- 本エージェントは Python 暗号資産トレードボットのアーキテクチャ改善・戦略検証・品質向上を支援する。
- コードリーディング結果と改善提案を継続的に集約し、重複解析を避けるために `analysis/` 以下へ中間成果物を保存する。
- 以後の解析では、まず既存ファイル (本ファイル / module_map.json / PROJECT_ANALYSIS_*.md) を参照し差分のみ追加する方針とする。

## 基本方針
1. 既存成果物を再利用し、再読込が必要な場合のみ対象ファイルを限定して読む。
2. 追加調査時は `PROJECT_ANALYSIS_YYYY-MM-DD.md` を日付単位で新規作成し、既存と重複する部分は「参照: 過去分析」と明記。
3. 変更点が生じたモジュールは `module_map.json` の該当セクションのみ更新 (全体再生成は避ける)。
4. 戦略ロジックやリスク計算の挙動差分は `strategies/CHANGELOG.md` 等へ分離を検討 (未作成)。

## 成果物一覧
- `ROLE_AND_INSTRUCTIONS.md` : 本ファイル。役割・運用ルール。
- `module_map.json` : クラス/関数/依存関係のマップ (静的解析ベース)。
- `PROJECT_ANALYSIS_2025-11-16.md` : 初期包括分析。以後は増分方式。

## 更新手順
1. 新規分析を行う前に既存 `module_map.json` を確認。
2. 新規モジュール追加時は差分部分のみ JSON に挿入。
3. 改善提案を実装したら `IMPROVEMENTS` セクションへ記録し、状態(提案 / 実装済 / 保留)を明示。
4. 実運用用とバックテスト用の差異が拡大した場合は `runtime_modes.md` を別途作成。

## 留意事項
- ログ・バックテスト結果など肥大化する成果物は原則集約/圧縮済み領域 (`logs/`) を参照。解析側にはメタ情報のみ記載。
- シングルトン利用箇所 (Logger, PriceDataManagement) の状態共有に伴う副作用へ注意する。
- 売買方向表記の不統一 (外部API: 'buy'/'sell', 内部: 'BUY'/'SELL') は設計上の整流候補。

## 次回以降の利用手順 (テンプレ)
```
1. analysis/ を確認
2. module_map.json の対象モジュールから再解析必要性を判定
3. 差分抽出 → PROJECT_ANALYSIS_<新日付>.md 作成
4. 必要なら module_map.json 部分更新
```

## 差分同期プロセス (M1 詳細)
1. `analysis/` 内成果物と `src/` コード構造を週次/変更イベント時に比較。
2. 新規/変更クラス検出時: `module_map.json` 部分更新 (全再生成回避)。
3. 戦略/リスク/設定の仕様差分は `PROJECT_ANALYSIS_<date>.md` に増分追記し過去参照リンク。
4. 乖離種別 (命名 / 引数順序 / 例外処理欠落 / ドキュ不足) をタグ化し Issue テンプレへ投入。
5. 解消後: 該当カテゴリステータスを `実装済` に遷移しコミットハッシュを記録。

## リファクタリングバックログ運用 (M3 詳細)
テンプレ (Issue):
```
タイトル: [C<ID>] <短い要約>
カテゴリ: C<ID>
背景: 現状問題 / 影響
ゴール: 期待される改善結果 (性能指標 / 可読性 / 安定性)
範囲: 変更予定ファイル / 非対象範囲
リスク: 破壊的変更リスク / 移行手順
検証: テストケース / メトリクス / ログ確認項目
```

優先度基準: (1) 安定性へ影響 (2) 戦略改善阻害 (3) 性能ボトルネック (4) メンテ性。

## 戦略改善ループ (M4 詳細)
1. 仮説設定: 指標 (勝率, PF, 最大DD, 日次シャープ) 目標値定義。
2. データ取得: 期間/ペア/足種を固定し再現性確保 (キャッシュ利用)。
3. バックテスト: 現行戦略 vs 変更案 (同期間) を並列実行。
4. 評価: 統計 (PnL曲線, 最大連敗, ボラ調整リターン) を比較し改善幅測定。
5. 採用判定: 最低改善閾値 (例: シャープ比 +0.2, 最大DD -10% 以上) 超過時に候補昇格。
6. 安全化: リスクユニット再計算 / 注文サイズ検証 / スリッページ耐性試験。
7. 実運用試験: 低サイズサンドボックス口座でステップ運用→問題なければ本番昇格。
